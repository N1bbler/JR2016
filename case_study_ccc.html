ChannelCatCharters.com:
Case Study

Project Profile

Type: Site Redesign

Category: Consumer-Facing Website

Description: The Channel Cat Charters website is the online presence for the Channel Cat ship located in Santa Barbara Harbor, SB, CA. The site is used as a sales and marketing tool, as well as to demonstrate the qualities and features of the Channel Cat charter experience to potential clientele. The business caters to a high-end audience and offers individualized and highly personalized service. The (now former) site was launched in 2007. While it served them well for many years, it was built with yesterday's technology (Flash, Java applets) and was no longer working for them or many of their clients. This resulted in lost booking opportunities, as well as having to guide clients through the site, which was expensive in terms of time allocation. Previously, the site did not include any analytics solution, so measuring these lost opportunities was impossible.

Purpose of the site:

  • For clientele to see the beauty, size, and overall grandeur of the ship and then to contact the Charter Agent for a booking
  • For Charter Agent and other stakeholders to use the site as a sales and marketing tool

Goals for the redesign:

  1. Make Virtual Tours work across devices, the tours are a big selling point
  2. Make Video interview with Extreme Yachts television show work across devices
  3. Update the Directions to be more clear, requires custom map routes as the mailing address is not the same as the boarding location
  4. Make it easier for Users to navigate the content itself (not in tiny window frame)

Problems to Solve:

  • Site too small on modern monitors, difficult to navigate as content is shown in tiny window
  • Home page animation not working on mobile (Flash-based)
  • Virtual Tours not working on mobile, or in some Desktop browsers (Java-applet based)
  • Video tour not working on mobile (Flash-based)
  • Gallery not working on mobile (Flash-based), difficult to navigate
  • Site required walk-through guidance to use
  • Directions need to be reworked, but cannot use physical address

Technical Challenges:

  • Most Virtual Tour software tends to be Java based, produce Java applet output
  • Video content is embedded in Flash, only source available
  • Client can send images via email, but many initial images sent are low-resoluton thumbnail images

Preliminary Client notes:

  • Wants Big beautiful pictures
  • Believes "Less is more" with marketing verbiage, wants pictures to speak for themselves
  • Client is a Senior Citizen that struggles a bit with computers and technology; ease of use, large images and scalable text is key
  • Client is not interested in going the route of a booking and reservation system; as a high-end chartering business with an exclusive clientele, they hold the customer's hands through every step of the booking process.

Discovery, Research, Wireframes

  • Initial client consultation in order to:
    • Understand the business itself and purpose of the site
    • Understand the problems they are facing with the site
    • Understand their goals for the site redesign
    • Understand their clientele, as well as their goals and needs
    • Understand their internal use cases for the site
    • Establish delivery of content and assets
    • Establish any parameters or limitations
    • Estimate time to completion, explain process, and address administrative matters
  • Perform Site Audit to understand current site, content, technical implementation and to reinforce goals
  • Analyze current workflow and information architecture
  • I then performed Competitive Analysis in order to:
    • Be aware of competing Charter companies in Santa Barbara
    • Evaluate the pros and cons of competitive sites
    • Understand where the bar is set, in order to be sure we overshoot it
  • Reviewed current and new site content for awareness and inspiration
  • Defined initial User Profiles / Personas, as well as simple Use Cases
  • Engage in initial ideation and wireframe sessions (analog) to quickly define potential solutions and approach
  • Modelled different Call-To-Action (CTA) approaches
  • Define and explain thoughts on Navigational Model changes (old site to new, responsive site)

Communicate, Validate, Iterate

  • Follow up client consultation in order to:
    • Validate findings from Competitive Analysis (Client agreed with findings)
    • Validate initial User Personas and Use Cases (they needed refining, these are rarely exact on the first attempt)
    • Validate initial wireframe concepts, walk them through and explain wireframes are like blueprints
    • Walked client through process of locating original files stored in Picassa which they use to manage photos, and how to determine file size and resolution of images via Windows Explorer
    • Obtained high-resolution photos and discussed content strategy with client
    • Discussed proposed navigational model with client
    • Explained concept of Call-To-Action and plan to have every page guide the User to Contact them, which ultimately is the primary goal of the site
